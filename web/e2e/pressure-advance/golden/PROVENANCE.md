# Pressure advance golden sample: Micron, Ambrosia ASA, provenance

## The physical sample

A pressure advance test coupon printed on the owner's Micron printer in Ambrosia ASA (dark test
lines on a contrasting base), generated by `pa/gcodeGenerator.ts` from the default test spec: 16
lines sweeping PA 0 to 0.06 (0.004 per line), slow segment 20 mm, fast segment 40 mm, slow speed
25 mm/s, fast speed 100 mm/s, line pitch 4 mm, nominal line width 0.45 mm. The default "Test
range" settings of a freshly opened PA page reproduce this spec exactly, so no settings form
field needs to be touched to reproduce the capture.

## The scan

- `pa_micron_asa.jpg`: one real flatbed scan, JPEG, an unmodified byte-for-byte copy of the
  original capture (`Micron_Ambrosia_ASA_PA.jpg`, owner's capture session), per the project's
  rule against resampling any scan image anywhere. The coupon was scanned face-down; the
  pipeline reads it as mirrored (flipped yes) at rotation 0, and measures the scan's own
  resolution geometrically as about 23.5 px/mm (a 600 dpi class scan).

The PA flow needs no scanner calibration: pressure advance is dimensionless and every distance
the analyzer uses is priced by the coupon's own fiducial geometry, so no card calibration is
seeded for this fixture.

## Approval tier: owner-reviewed

The owner tunes this printer and filament combination by hand and knows its correct pressure
advance lies in the band 0.024 to 0.032. The pipeline's answer on this scan was reviewed against
that band and judged correct from that domain knowledge; no correction was applied back to the
printer in this capture session, so the hardware-validated tier does not apply. Only this single
600 dpi class scan exists for this sample; a second, lower-resolution scan of the same coupon is
an open gap against the multi-DPI convention and should be added when the owner rescans it.
